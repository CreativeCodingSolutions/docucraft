12:T9a1,<p class="leading-relaxed text-muted-foreground">~15% of PRs in popular open-source repos have no description at all. Another ~10% are one-liners that don't explain the *why*. I analyzed 100+ PRs across repos like <code>softprops/action-gh-release</code> (5.6k stars) and <code>docker/build-push-action</code> (5.3k stars) — roughly 1 in 7 PRs was empty.</p>
<!-- -->
<p class="leading-relaxed text-muted-foreground">Examples from the data:</p>
<li class="ml-6 list-disc text-muted-foreground"><a href="https://github.com/softprops/action-gh-release/pull/787" target="_blank" rel="noopener noreferrer" class="text-primary underline underline-offset-4 hover:text-primary/80">#787</a>: &quot;Update the latest release&quot; — 6 files, +90/-35, description: blank</li>
<li class="ml-6 list-disc text-muted-foreground"><a href="https://github.com/docker/build-push-action/pull/1550" target="_blank" rel="noopener noreferrer" class="text-primary underline underline-offset-4 hover:text-primary/80">#1550</a>: &quot;mention Docker GitHub Builder in the README&quot; — description: blank</li>
<!-- -->
<p class="leading-relaxed text-muted-foreground">I built DocuCraft to fix this. It's a GitHub Action that reads the diff on every PR and writes a structured description: summary, file-by-file breakdown, change categories, and labels. No API keys needed, no signup.</p>
<!-- -->
<p class="leading-relaxed text-muted-foreground">Check out the live demo at <a href="https://github.com/CreativeCodingSolutions/docucraft/pull/4" target="_blank" rel="noopener noreferrer" class="text-primary underline underline-offset-4 hover:text-primary/80">#4 on the DocuCraft repo</a> — it auto-generated its own PR description:</p>

<pre class="overflow-x-auto rounded-lg border bg-muted p-4 text-sm"><code># .github/workflows/docucraft.yml
name: DocuCraft
on: pull_request
permissions: { contents: read, pull-requests: write }
jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: CreativeCodingSolutions/docucraft@v1
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}</code></pre>
